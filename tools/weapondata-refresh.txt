Weapon data refresh notes
=========================

Source of truth
---------------
The app now loads weapon data from the checked-in file:

    weapons\weapondata.csv

That file is the source of truth for the app. The old Google Sheets export is
now just an import source, not the runtime dependency.


Why this changed
----------------
Previously the app fetched a live CSV from Google Sheets at runtime. That made
the site dependent on an external spreadsheet and made it awkward to add local
fields like `RPM` for TTK work.

Keeping the CSV in-repo makes the app self-contained and lets future edits be
reviewed and committed alongside code changes.


Refresh workflow
----------------
1. Download the latest Google Sheets CSV export from:

       https://docs.google.com/spreadsheets/d/e/2PACX-1vTeLqZ5-maEmzrM6SUDMRXpHEhV0tQImiBdgMCil9lSA11IiY_nGdamE54W7DAiSXn1XuJljdF4P537/pub?gid=0&single=true&output=csv

2. Replace `weapons\weapondata.csv` with that export.

3. Generate a fresh `filediver` entity/component dump:

       Set-Location C:\GitHub\filediver
       go run .\cmd\tools\components\entity-component-settings-json-dumper > C:\temp\entity-component-settings.json

4. Merge RPM values from the dump into the checked-in CSV:

       Set-Location C:\GitHub\supercalc
       python .\tools\merge_weapon_rpm_from_filediver.py --dump C:\temp\entity-component-settings.json --csv .\weapons\weapondata.csv

5. Review unresolved rows and odd values.

6. Use wiki raw JSON only as a fallback/audit source for gaps that `filediver`
   does not cover cleanly. The current best raw source is:

       https://helldivers.wiki.gg/wiki/Module:Decodedata-Weapons/data.json?action=raw

   You can pass it directly to the merge helper when you want wiki backfills:

       python .\tools\merge_weapon_rpm_from_filediver.py --dump C:\temp\entity-component-settings.json --csv .\weapons\weapondata.csv --wiki-url https://helldivers.wiki.gg/wiki/Module:Decodedata-Weapons/data.json?action=raw

7. Run tests before committing.


RPM column notes
----------------
The loader already recognizes these header variants:

- `RPM`
- `ROF`
- `Fire Rate`
- `Rate of Fire`
- `Rounds per Minute`

The preferred current path is:

- keep `RPM` in `weapons\weapondata.csv`
- populate it automatically from `filediver` where possible
- use wiki data only for backfills or validation

The first pass should accept a clear/default scalar RPM even if it does not
perfectly model bolt cycling or animation timing. Continuous beam and spray
weapons can stay blank until they have a better cadence model.


Patch metadata
--------------
Because the app now loads a stable local filename, it no longer relies on the
Google Sheets response headers to infer patch metadata.

If you temporarily point the app at a versioned filename, the loader can still
extract a version token like `1.006.003` from the filename, but the stable
checked-in path should remain `weapons\weapondata.csv`.
