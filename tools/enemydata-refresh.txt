Enemy data refresh notes
========================

Purpose
-------
This repository's checked-in `enemies\enemydata.json` is curated review output,
not a direct export from the raw Helldivers 2 `data\` folder.

The repeatable workflow is now:

1. Dump entity/component data from `filediver`
2. Flatten it with `tools\flatten_entity_component_settings.py`
3. Parse it with `tools\parser_faction_units.py`
4. Review it with `tools\compare_enemydata.py`
5. Curate the checked-in file instead of blindly overwriting it


Important caveat
----------------
`filediver`'s `entity-component-settings-json-dumper` does detect the local
Helldivers 2 install and load the string table, but the actual entity/component
payload comes from embedded datalibrary blobs such as
`datalibrary\generated_entities.dl_bin.gz`.

Practical consequences:

- A complete local `data\` folder is still useful for strings and lookup data
- Missing units are not explained by an incomplete local install alone
- A stale `filediver` datalibrary snapshot can still lag the latest game build
- A current local `filediver` checkout is necessary, but not sufficient

Step 1: dump entity component settings
--------------------------------------
PowerShell:

    Set-Location C:\GitHub\filediver
    & 'C:\Program Files\Go\bin\go.exe' run .\cmd\tools\components\entity-component-settings-json-dumper > C:\temp\entity-component-settings.json


Step 2: flatten the dump into parser_faction_units.py input shape
-----------------------------------------------------------------
PowerShell:

    Set-Location C:\GitHub\supercalc
    python .\tools\flatten_entity_component_settings.py --input C:\temp\entity-component-settings.json --output C:\temp\entity-component-settings-flat.json

This script handles both the older direct-component shape and the newer nested
`components.*` shape, plus these key fallbacks:

- top-level `content/fac_*` key when already resolved
- `UnitComponentData.unit_path`
- `LocalUnitComponentData.unit_path`
- synthesized `content/fac_<bucket>/__unresolved__/<hash>` fallback from
  `FactionComponentData`
- `loc_name_upper` fallback when `loc_name` is missing


Step 3: run the existing parser
-------------------------------
PowerShell:

    Set-Location C:\GitHub\supercalc
    python .\tools\parser_faction_units.py --input C:\temp\entity-component-settings-flat.json --output C:\temp\enemydata-from-filediver.json


Step 4: review before checking in
---------------------------------
PowerShell:

    Set-Location C:\GitHub\supercalc
    python .\tools\compare_enemydata.py --generated C:\temp\enemydata-from-filediver.json --report C:\temp\enemydata-report.json --normalized-generated C:\temp\enemydata-normalized.json

Do not blindly overwrite `enemies\enemydata.json`. The generated file is review
input, not the checked-in source of truth.

The comparison helper:

- normalizes `MainCap` bool vs int noise
- treats missing `ExMult` as default `1`
- reports added units, missing/manual-only units, and stat-changed units
- reports name-only zone churn separately
- carries forward existing readable zone names only when a generated hashed zone
  has a unique safe signature match

Why the first pass looked like 11 units were missing
----------------------------------------------------
The initial gaps were mostly adapter issues, not missing game files:

- Many units exist in the raw `filediver` dump but use unresolved hashed
  top-level keys instead of `content/fac_*`
- `Shredder Tank` has `loc_name_upper` and `loc_name_plural`, but no `loc_name`
- `Heavy Fusion Cannon` has a usable `UnitComponentData.unit_path`, but no
  `FactionComponentData`

In other words, the raw dump often had the data; the missing units came from
assuming that a valid entry must already have both:

- a resolved `content/fac_*` top-level key
- a populated `loc_name`

Recommended workflow for future refreshes
-----------------------------------------
1. Update `filediver` to latest GitHub HEAD
2. Re-run the entity dump
3. Run `tools\flatten_entity_component_settings.py`
4. Run `tools\parser_faction_units.py`
5. Run `tools\compare_enemydata.py`
6. Manually review:
   - `Factory Strider Gatling Gun`
   - any newly surfaced units
   - `stat_changed_units`
   - `name_only_units`
   - any zones whose generated names are still opaque hashes
7. Only then update the checked-in file
